### `loop()` — メインループ

> ※ loop() は「状態ごとに何をするか」だけ書く。細かい処理は各関数に任せる。

```
【処理の流れ】

＜毎ループ実行すること＞
  - 入力を読む（readButton(), readSensor() などを呼ぶ）
  - 現在時刻を取得: now = millis()

＜currentState が 0（待機中）のとき＞
  - センサー値を監視する
  - 検知条件を満たしたら → currentState = 1

＜currentState が 1（動作中）のとき＞
  - メイン処理を行う
  - 終了条件を満たしたら → currentState = 2

＜currentState が 2（完了）のとき＞
  - 完了表示をする
  - リセットボタンが押されたら → currentState = 0

＜currentState が 3（エラー）のとき＞
  - エラー表示をする / リセットを待つ
```

**↓ 自分の loop() を設計してください**

```
＜毎ループ実行すること＞
- ボタン入力を読む（readButton() を呼ぶ）

＜currentState が 0（待機中）のとき＞

- 終了音を鳴らす
- LED を消灯する
- 処理後、currentState = 0 に戻す

```

## 3. 重要ロジックの詳細設計

### 3-1. チャタリング防止（デバウンス処理）

> ※ ボタンを使う場合は必ず設計してください。

```
【考え方】
  ボタンが押されたとき、50ms 以内の連続入力は「同じ1回の押下」として無視する。

【処理の流れ】
  1. ボタンのデジタル値を読む（digitalRead）
  2. 前回確定した時刻（lastDebounceTime）からの経過時間を計算する
  3. 経過時間 < DEBOUNCE_DELAY（例: 50ms）→ 無視する
  4. 経過時間 ≥ DEBOUNCE_DELAY → ボタンの状態として確定する
  5. lastDebounceTime を更新する

【必要な変数（Section 1 に追加済みか確認）】
  lastDebounceTime : unsigned long   // 前回確定した時刻
  DEBOUNCE_DELAY   : const int = 50  // チャタリング判定時間（ms）
```
【考え方】
ボタンが押されたとき、短時間の揺れを1回の押下として扱う。

【処理の流れ】
1. ボタンのデジタル値を読む
2. 前回の確認時刻からの経過時間を調べる
3. 50ms 未満なら無視する
4. 50ms 以上なら押下状態として扱う
5. lastDebounceTime を更新する

【必要な変数】
- lastDebounceTime : unsigned long
- DEBOUNCE_DELAY   : const int = 50

### Q1: 実装上の問題確認

> 「この詳細設計書に書いた関数と処理フローをもとに Arduino でコードを書きます。バグになりやすい箇所・処理の抜け・型の問題はありますか？」

**AIの回答（要約）：**
**AIの回答（要約）：**
状態遷移の条件, ボタン入力の扱い, 型の整合性 がバグになりやすいです。


ボタンの押下判定

INPUT_PULLUP のため、押したときは LOW になります。
HIGH/LOW の反転を見落とすと、押していないのに押下扱いになる可能性があります。
デバウンス処理

lastDebounceTime を使うなら、どのタイミングで更新するか を明確にしないと誤動作します。
「押下確定時のみ更新」なのか「読み取りごとに更新」なのかを統一する必要があります。
状態遷移の重複

loop() と readButton() の両方で状態変更を行うと、処理が二重になる恐れがあります。
状態変更は1か所に集約した方が安全です。
focusStarted の役割が曖昧

現状だと currentState と役割が重なっており、不要になる可能性があります。
使わないなら削除した方が設計がすっきりします。
型の確認

millis() は unsigned long なので、時刻保存用変数も必ず unsigned long にする必要があります。
int にすると長時間動作時に不具合が出る可能性があります。
終了処理の扱い

currentState = 3 の後にすぐ 0 に戻すなら、終了音が十分鳴らないことがあります。
delay() を使わずに音を鳴らす時間を確保する方法を考えるとよいです。
**対応した内容：**
ボタンは LOW で押下と明記
状態変更は loop() 側に集約
focusStarted は削除候補にする
時刻管理変数は unsigned long に統一
終了音の表示時間を確保する処理を追加検討
