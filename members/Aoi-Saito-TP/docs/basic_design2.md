### 1-2. 状態遷移（ガジェットの「ふるまい」設計）

> ※ ガジェットが「どんな状態を持つか」を定義します。
> この設計が loop() の骨格になります。

**記入例：**
```
[電源ON / 初期化]
  ↓（初期化完了）
[待機中] ──（センサー検知）──→ [計測中] ──（閾値超え）──→ [警告中]
   ↑                                                            │
   └────────────────────（リセットボタン）────────────────────────┘
```

**↓ 自分の状態遷移を書いてください**
```

  ↓（初期化完了）

   ↑                                           │
   └────────────（距離が30cmより遠い）───────┘


```

**各状態の意味：**

| 状態名 | この状態のとき、何をしているか | LEDやブザーなど出力の様子 |
|:--|:--|:--|
| （例）待機中 | センサー値を100msごとに監視する | 緑LED：500msで点滅 |
|初期化  |ピン設定やライブラリ準備を行う|ブザーは鳴らさない
|待機・計測中  | 距離を定期的に測定する |30cm超ならブザー停止  |
|警告中|距離に応じた音を出す|近いほど高い音で鳴る

### 2-3. タイミング設計（delay vs millis）

> ※ `delay(N)` はN ms間、**すべての処理**が止まります。
> 複数の処理を並行させたい場合は `millis()` を使います。

| 処理 | 必要な周期 | delay / millis | 判断の理由 |
|:--|:--|:--|:--|
| （例）LED点滅 | 500ms | **millis** | ボタン監視と並行するため |
| （例）センサー読み取り | 100ms | **millis** | 常時監視が必要なため |
|  |  |  |  |
|  |  |  |  |
処理	必要な周期	delay / millis	判断の理由

距離測定	100ms程度	millis	定期的に更新した方が自然なため
シリアル表示	100ms程度	millis	見やすさを保つため
ブザー制御	距離測定に合わせる	millis	センサー更新と同期させるため
初期化後の待ち	1秒程度	delay	起動直後だけなら問題が少ないため
> [!CAUTION]
> `delay()` を使う場合、その間はボタン入力もセンサー読み取りも**すべて止まります**。

## 7. AIレビュー記録

> グループレビューの前に必ず実施してください。

### Q1: 設計の妥当性確認

> 「この基本設計書をもとに Arduino UNO R3 でコードを書きます。ピン競合・ライブラリ競合・状態遷移の抜け・delay() の問題点はありますか？」

**AIの回答（要約）：**
ピン競合はなし。HC-SR04 とブザーで別ピンを使用しており、Serial 用の D0/D1 も使ってない。
ライブラリ競合も特にありません。超音波センサーは自前処理でも実装可能。
状態遷移は「初期化 → 計測中 → 警告中」で最小限にまとまっていますが、異常値時に「停止状態へ戻す」流れを明確にすると安全。
delay() は起動直後だけなら問題ありませんが、通常処理で使うと距離変化への反応が遅れる。
**対応した内容：**
ピンは D12 / D11 / D8 に分離して設計済み
Serial 競合を避けるため D0 / D1 は未使用
異常値時はブザー停止にする方針を明記した
delay() は初期化後の短い待ちのみに限定し、通常制御は millis() を使用する方針にした
---

### Q2: チャタリング・delay・ノイズの確認

> 「この設計にチャタリング対策は含まれていますか？delay() を使っている箇所で問題はありますか？アナログセンサーを使う場合、ノイズ対策は必要ですか？」

**AIの回答（要約）：**
この作品ではボタン入力を使っていないため、チャタリング対策は必須ではない。
delay() は初期化後の待ちにのみ使う想定なので、通常動作への影響は限定的。
アナログセンサーは使用していませんが、もし値が揺れる場合は平均化やしきい値の幅を設けると安定する。
超音波センサーでも誤読はあり得るため、0cm や極端な値を異常値として無視する設計は有効。

**対応した内容：**
ボタン機能を入れないことでチャタリング問題を回避した
delay() は起動直後の1回だけに限定した
センサー異常値は無視し、前回値または停止にする方針を入れた
必要なら測定値の簡易平均化を追加検討することにした

---
