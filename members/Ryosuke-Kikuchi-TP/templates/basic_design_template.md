## 1. システム全体像

### 1-1. ブロック図（入力 → 処理 → 出力）

> ※ Arduinoを中心に、「何が入力で何が出力か」を整理します。  
> ASCII図または別紙の写真を貼り付けてください。

**記入例：**
```
【入力】               【Arduino UNO R3】          【出力】
超音波センサー ────→┐                       ┌──→ LED（赤・緑）
タクトスイッチ ────→┤    状態管理・制御      ├──→ パッシブブザー
                    └─────────────────────┘
```

**↓ 自分のシステムを書いてください**
```
【入力】               【Arduino UNO R3】         　【出力】
Membrane Switch ────→┐                       ┌──→ LED（青）
                     ┤  タイマー状態管理・制御  ├──→ LED（赤）
                     └─────────────────────┘
                                  ├──→ パッシブブザー
                                  └──→ シリアルモニタ表示（任意で7セグ追加）


```

### 1-2. 状態遷移図（システムの「ふるまい」）

> ※ ガジェットが「どんな状態を持つか」を定義します。  
> 状態（○）と、状態が切り替わる条件（矢印の横）を書いてください。

**記入例：**
```
[電源ON]
  ↓ 初期化完了
[待機中] ─── センサー検知 ───→ [動作中] ─── 終了条件 ───→ [完了]
  ↑                                 │
  └──────── リセット ────────────────┘
```

**↓ 自分の状態遷移を書いてください**
```
[電源ON]
  ↓
[待機中] ─── スタート押下（時間設定済み） ───→ [計測中] ─── 残り時間0秒 ───→ [終了通知]
  ↑                     │                          │
  └── リセット長押し  ───┴── 停止押下で待機へ　───────┘

[終了通知] ─── 通知完了（ブザー5秒+赤LED点滅（1秒に5回）） ───→ [待機中]

```

**状態の意味を説明してください：**

| 状態名 | この状態のとき、何をしているか |
|:--|:--|
| （例）待機中 | センサーの値を監視しながら、LEDを一定間隔で点滅 |
| 待機中 | 青LED消灯。キー入力で時間設定を受け付け、設定値はシリアルモニタに表示する。 |
| 計測中 | 青LED点灯。1秒ごとに残り時間を減算し、残り時間はシリアルモニタに表示する。 |
| 終了通知 | 赤LEDを1秒に5回点滅し、ブザーを5秒鳴らす。通知後は待機中へ戻る。 |

## 2. ハードウェア設計

### 2-1. 部品一覧と接続ピン

> ※ 使う部品とArduinoのどのピンに繋ぐかを決めます。  
> デジタル（D2〜D13）/ アナログ（A0〜A5）/ 通信（I2C: A4/A5, SPI: D10〜D13）

| 部品名 | 型番・種類 | Arduinoピン | 入力/出力 | 備考 |
|:--|:--|:--|:--|:--|
| Membrane Switch Module | 4x4キーパッド | D2, D3, D7, D8, A0, A1, A2, A3 | 入力 | 時間設定・開始・停止・リセットに使用 |
| LED（青） | 5mm LED | D9 | 出力 | 220Ω抵抗と直列 |
| LED（赤） | 5mm LED | D10 | 出力 | 220Ω抵抗と直列 |
| パッシブブザー | Passive Buzzer Module | D11 | 出力 | 終了通知音（5秒）に使用 |
| （任意）4Digit 7-Segment Display | 4桁7セグ（TM1637なし） | 追加時に別途割り当て | 出力 | 時間表示を物理表示にしたい場合のみ実装 |
| （任意）74HC595 | シフトレジスタ | 追加時に別途割り当て | 出力 | 7セグのa,b,c,d,e,f,g,dpを制御 |
| （任意）2N2222（4個） | NPNトランジスタ | 追加時に別途割り当て | 出力 | 4桁7セグの桁選択（電流増幅）に使用 |
| 抵抗（2個） | 220Ω | なし | なし | 青/赤LEDの電流制限 |
| （任意）抵抗（8個） | 220Ω | なし | なし | 7セグ各セグメントの電流制限 |
| （任意）抵抗（4個） | 1kΩ | なし | なし | 2N2222ベース抵抗 |
| ブレッドボード / ジャンパ線 | 配線材 | なし | なし | 回路作成に使用 |

> [!TIP]
> **ピン競合チェック：** I2C（A4/A5）・SPI（D10〜D13）は複数部品で共用します。  
> デジタルピン（D2〜D9）は原則1部品1ピンです。D0/D1はシリアル通信に使われるため避けること。

### 2-2. 使用する抵抗・回路メモ

> ※ 抵抗値の計算や、注意が必要な接続を記録してください。

| 用途 | 計算・メモ |
|:--|:--|
| LED電流制限抵抗 | 青/赤LED: (5V-2.0V)/0.01A=300Ω程度。標準値220Ωを使用（十分な明るさを優先）。 |

### 3-2. 関数一覧（役割の定義）

> ※ プログラムを小さな「処理の部品（関数）」に分割します。  
> **ここではコードを書かず、「何をする関数か」だけ日本語で書いてください。**  
> 具体的な実装は詳細設計書（detailed_design.md）で行います。

| 関数名 | 何をする関数か（1〜2行で） | 引数（入力） | 戻り値（出力） |
|:--|:--|:--|:--|
| `setup()` | Arduinoの初期化・ピンモード設定 | なし | なし |
| `loop()` | 状態に応じて各関数を呼び出すメインループ | なし | なし |
| （例）`readSensor()` | 超音波センサーの距離を測って返す | なし | int（cm） |
| （例）`updateLED(state)` | 現在の状態に合わせてLEDの点灯パターンを更新 | int state | なし |
| `readKeypad()` | キーパッド入力を読み取り、時間設定/開始/停止/リセット操作を判定する | なし | char（押下キー） |
| `updateTimer()` | `millis()`ベースで残り時間を更新し、0秒到達を判定する | なし | bool（終了到達） |
| `showTimeSerial(mm, ss)` | 設定時間と残り時間をシリアルモニタへ表示する | int mm, int ss | なし |
| （任意）`displayTime(mm, ss)` | 74HC595と桁選択を使って4桁7セグに時刻を表示する | int mm, int ss | なし |

### 3-3. タイミング設計（delay vs millis）

> ※ 複数の処理を同時に動かす場合、`delay()`を使うと他の処理が止まります。  
> 各タイマーが `delay` か `millis` かを事前に決めておきます。

| 処理 | 周期 | delay / millis | 理由 |
|:--|:--|:--|:--|
| （例）LED点滅 | 500ms | **millis** | ボタン監視と並行するため |
| （例）センサー読み取り | 100ms | **millis** | 常時監視が必要なため |
| キーパッド入力スキャン | 10〜20ms | **millis** | チャタリング対策と応答性を両立するため |
| タイマー減算 | 1000ms | **millis** | 1秒単位で正確に減算するため |
| 終了通知（ブザー5秒/赤LED点滅） | 100〜200ms更新 | **millis** | 通知中も停止キー入力を受け付けるため |
| （任意）7セグ桁更新（マルチプレックス） | 2〜5ms/桁 | **millis** | 点灯むら・ちらつきを防ぎつつ他処理を止めないため |

## 4. 異常系・エラー処理の方針

> ※ 要件定義書の「非機能要件（3-4）」を具体化します。  
> 詳細な実装方法は詳細設計書で書きます。ここでは「どう検知し、どう対応するか」の方針だけ。

| 異常ケース | 検知方法 | 対応方針 |
|:--|:--|:--|
| キー入力が不正（未定義キー） | 許可キー集合（数字/開始/停止/リセット）に含まれない入力を検出 | その入力を無視し、表示と状態は変更しない |
| 設定時間が範囲外（0秒/99分59秒超） | 入力確定時に範囲チェックを行う | 範囲外は受け付けず、直前の有効値を維持する |
| （任意）7セグ表示の桁欠け/不点灯 | 桁更新周期の監視とデバッグ表示で異常を検知 | 配線・抵抗値を再確認し、異常時は表示を消灯して安全側に倒す |

## 6. AIレビュー記録（基本設計版）

### Q1: 構造の妥当性確認
> 「この基本設計書をもとに Arduino UNO R3 でコードを書くとき、  
> ピン不足・ライブラリの競合・状態設計の漏れはありますか？」

**AIの回答（要約）：**
- 必須機能だけなら7セグなし構成の方が回路・実装ともに簡単で、ピンにも余裕が出る。
- 時間表示はシリアルモニタで代替可能で、後から7セグを任意機能として追加できる。
- `delay()`中心だと入力取りこぼしが起きるため、状態遷移と通知処理は`millis()`で管理するのが安全。

**対応した内容：**
- 必須構成をLED・ブザー・キーパッド中心に整理し、7セグ関連部品は任意として分離した。
- 時間表示の基本方針をシリアルモニタ表示に変更した。
- タイミング設計を`millis()`ベースで統一した。

---

### Q2: 詳細設計への引き継ぎ確認
> 「この基本設計書から詳細設計に進むとき、  
> まだ決まっていない・詳細化が必要な部分はどこですか？」

**AIの回答（要約）：**
- キーパッドのキー割り当て（数字/開始/停止/リセット）を詳細設計で確定する必要がある。
- （任意）7セグが共通アノード/共通カソードのどちらかにより、2N2222接続極性と点灯ロジックが変わる。
- ブザーの簡易メロディーは周波数テーブルと再生手順を詳細化する必要がある。

**対応した内容：**
- 詳細設計へ引き継ぐ未決事項として、キー割り当て・表示極性・メロディー仕様を整理した。
