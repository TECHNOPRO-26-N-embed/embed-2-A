## 1. システム全体像

### 1-1. ブロック構成（入力 → 処理 → 出力）

> ※ Arduinoを中心に「何が入力で何が出力か」をテキストの図で整理します。

**記入例：**
```
【入力】                 【Arduino UNO R3】            【出力】
タクトスイッチ ─────→┐                         ┌──→ LED（赤・緑）
超音波センサー ─────→┤   状態管理・判定処理    ├──→ パッシブブザー
                   └─────────────────────────┘
```

**↓ 自分のシステムを書いてください**
```
【入力】                 【Arduino UNO R3】            【出力】
ジョイスティック ────→|   判定・制御処理   |───→ パッシブブザー
                         |                    |───→ 白色LED
                         └────────────────────┘
```

補足説明（図で表現しきれない場合）：
ジョイスティックから入力された値をArduinoで処理し、音程および音の長さを制御する。
また、ランダムに生成された目標音とプレイヤーの入力音を比較し、結果をブザーとLEDでフィードバックする。

### 1-2. 状態遷移（ガジェットの「ふるまい」設計）

> ※ ガジェットが「どんな状態を持つか」を定義します。
> この設計が loop() の骨格になります。

**記入例：**
```
[電源ON / 初期化]
  ↓（初期化完了）
[待機中] ──（センサー検知）──→ [計測中] ──（閾値超え）──→ [警告中]
   ↑                                                            │
   └────────────────────（リセットボタン）────────────────────────┘
```
                                                   │
                                                   └──→ [確認] ───→ [提出]

**↓ 自分の状態遷移を書いてください**
```
[電源ON/初期化]
  ↓（初期化完了）                                            
[待機中] ──（ジョイスティック押す）──→ [出題] ─(再生)→[入力]─(再生)
   ↑                                            　　　　　　 │
                                                   ↑ ──────」(ボタン押下で提出)
   └─────────（次の問題）──────[結果表示]←────[判定]←─────────┘
```

**各状態の意味：**

| 状態名 | この状態のとき、何をしているか | LEDやブザーなど出力の様子 |
|:--|:--|:--|
| 待機中 | スタート待ち | LED消灯 |
| 出題 |ランダムに生成した音を再生|ブザーが鳴る|
| 入力 | ジョイスティック操作で鳴らす音を調整 | ブザーがリアルタイムに鳴る |
| 判定 | 音の比較処理 | なし |
| 結果表示 | 正解または不正解の結果を表示する | 正解：LED点灯＋高音、不正解：LED点滅＋低音 |

> [!TIP]
> すべての状態から「抜け出せるルート」があるか確認してください。
> 「このボタンを押せば必ずリセットに戻る」が保証されていますか？

### 2-3. タイミング設計（delay vs millis）

> ※ `delay(N)` はN ms間、**すべての処理**が止まります。
> 複数の処理を並行させたい場合は `millis()` を使います。

| 処理 | 必要な周期 | delay / millis | 判断の理由 |
|:--|:--|:--|:--|
| LED点滅 | 500ms | millis | 他処理と並行 |
| ジョイスティック読取 | 50ms | millis | 応答性重視 |
| 音再生(出題・入力) | 必要時 | millis | 入力処理と並行するため |
|判定音・結果音|100msぐらい|delay|一時的な演出|
| 長押し判定 | 1秒 | millis | 確認と提出を区別するため |

> [!CAUTION]
> `delay()` を使う場合、その間はボタン入力もセンサー読み取りも**すべて止まります**。
